# 📢 Обновления AVTORGRAF v2.1 (Streaming + UI улучшения)

## 🎯 Краткое резюме

Добавлена поддержка **streaming режима** для плавного отображения ответов и значительно улучшен UI для лучшего восприятия информации.

---

## ✨ Основные изменения

### 1. 🚀 Backend — Streaming поддержка

**Файл:** `src/avtorgraf/presentation/http_server.py`

- ✅ Добавлен метод `_handle_chat_stream()` для Server-Sent Events (SSE)
- ✅ Ответ отправляется по предложениям в реальном времени
- ✅ Поддержка обратной совместимости (non-streaming режим работает как раньше)
- ✅ Метаданные и ссылки отправляются в потоке

**Как работает:**
```python
# Клиент отправляет:
{ "stream": true }

# Сервер отправляет события:
data: {"type": "metadata", "session_id": "..."}
data: {"type": "chunk", "content": "Первое предложение."}
data: {"type": "chunk", "content": "Второе предложение."}
data: {"type": "references", "references": [...]}
data: {"type": "done"}
```

### 2. 🎨 Frontend — Streaming поддержка

**Файл:** `static/app.js`

- ✅ Функция `ask()` теперь использует EventSource для streaming
- ✅ Добавлены функции `handleStreamingResponse()` и `finalizeAssistantMessage()`
- ✅ Сообщения ассистента создаются динамически
- ✅ Ссылки на документы извлекаются и отображаются под ответом

**Улучшения:**
- Пользователь видит ответ по мере его получения
- Не нужно ждать пока весь ответ загружается
- Плавное добавление текста в чат

### 3. 📚 UI — Ссылки на документы

**Файл:** `static/styles.css` + `static/app.js`

- ✅ Ссылки на документы теперь отображаются **под ответом мелким шрифтом**
- ✅ Убрана отдельная боковая панель "Нормативные ссылки"
- ✅ Каждый ответ имеет свои ссылки

**Внешний вид:**
```
┌─────────────────────────────────────────┐
│ Вопрос: Как подписать АОСР?             │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ Ответ: Согласно нормам необходимо...    │

│ [СН 1.03.01-2019, п. 3.4]              │
│ [СТБ 1937-2022, раздел 2]              │

```

### 4. 📋 Улучшение отображения документов

**Файл:** `static/app.js`

- ✅ Функция `refreshDocuments()` теперь показывает **список файлов вместо JSON**
- ✅ Красивое форматирование с статусами
- ✅ Легко читать какие документы загружены

**Было:**
```json
{"documents": [...], "pipeline": {...}}
```

**Теперь:**
```

3. Инструкция 1-2020.pdf [processed]
```

### 5. 💡 UI — Примечания под кнопками

**Файл:** `static/index.html` + `static/styles.css`

- ✅ Добавлены подсказки под выбором ролей (ГИП, АН, ТН)
- ✅ Добавлены подсказки под выбором режима поиска
- ✅ Мелкий шрифт, не мешает интерфейсу

**Примечания:**
```
ГИП — главный инженер проекта | АН — авторский надзор | ТН — технический надзор
Выбранная роль добавляется в контекст вашего вопроса для более точных ответов

Mix — оптимальный баланс | Hybrid — комбинированный поиск | Global — полный граф знаний
Local — локальный контекст | Naive — базовый поиск
```

## 📊 Технические детали

### Streaming API

**Запрос:**
```javascript
fetch("/api/chat", {
  method: "POST",
  body: JSON.stringify({
    question: "...",
    session_id: "...",
    mode: "mix",
    stream: true  // ← Включает streaming
  })
})
```

**Ответ (Server-Sent Events):**
```
Content-Type: text/event-stream

data: {"type": "metadata", "session_id": "abc123", "elapsed_ms": 2500}

data: {"type": "chunk", "content": "Согласно"}

data: {"type": "chunk", "content": " нормам РБ"}

data: {"type": "references", "references": [{"title": "СН 1.03.01-2019, п. 3.4"}]}

data: {"type": "done"}
```

### Fallback механизм

Если сервер не поддерживает streaming, используется обычный JSON режим:
```javascript
{
  "session_id": "abc123",
  "answer": "Полный ответ...",
  "references": [...],
  "elapsed_ms": 2500
}
```

## 🎓 Использование

### Для обычного пользователя

1. Выберите роль (ГИП, АН или ТН)
2. Выберите режим поиска (Mix, Hybrid, etc)
3. Задайте вопрос
4. **Видите ответ в реальном времени** ← Новое!
5. Ссылки на документы отображаются внизу ← Новое!

### Для разработчика

Streaming включена по умолчанию. Чтобы отключить:

```javascript
fetch("/api/chat", {
  body: JSON.stringify({
    question: "...",
    stream: false  // Отключить streaming
  })
})
```
