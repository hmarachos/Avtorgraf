## 📦 Установка (обновление)

### Docker Compose (рекомендуется)

```bash
# Если вы уже используете v1:
docker compose pull
docker compose up --build

# Если первый запуск:
cp .env.example .env
docker compose up
```

**Приложение будет доступно:** http://localhost:8080

### Локальный запуск

```bash
# Для macOS/Linux:
PYTHONPATH=src python -m avtorgraf.main

# Для Windows:
set PYTHONPATH=src & python -m avtorgraf.main
```

Приложение будет доступно на http://localhost:8080

## 🎯 Первый запрос (пример)

### 1. Откройте http://localhost:8080 в браузере

### 2. Выберите роль (ГИП, АН или ТН)

### 3. Задайте вопрос, например:

> "Какие документы нужны для подписания акта освидетельствования скрытых работ?"

### 4. Получите форматированный ответ

**Раньше:**
```
Согласно СН 1.03.01-2019 нужны: документы, проект, сметы...
```

**Теперь:**
```
## Документы для АОСР

Согласно СН 1.03.01-2019 нужны:

- Проектная документация
- Рабочая документация
- Смета затрат
- ...
```

## 📱 Функции

### Быстрые вопросы

Нажимайте на кнопки для шаблонных вопросов:
- 🔄 Замена материалов
- 📋 Скрытые работы
- 🏛️ Визит инспектора
- 📝 Запись в журнал

### Роли пользователя

- **ГИП** — главный инженер проекта
- **АН** — авторский надзор
- **ТН** — технический надзор

Выбранная роль добавляется в контекст для каждого запроса.

### Режимы поиска

- **mix** — оптимальный баланс (рекомендуется) ⭐
- **hybrid** — комбинированный поиск
- **global** — по всему графу знаний
- **local** — локальный контекст
- **naive** — базовый поиск

## 🔧 Конфигурация

### Основные переменные окружения

```env
# Адрес и порт приложения

# LightRAG сервер
LIGHTRAG_BASE_URL=http://46.17.105.48:9621
LIGHTRAG_API_KEY=ваш-ключ-здесь

# Режим поиска по умолчанию
DEFAULT_QUERY_MODE=mix
```

Полный список см. в `.env.example`

## 🚨 Решение проблем

### Проблема: Приложение не запускается

```bash
# Проверьте что Python установлен
python --version  # Должно быть 3.12+

# Проверьте переменные окружения
cat .env

# Проверьте что LightRAG доступен
curl http://46.17.105.48:9621/api/health
```

### Проблема: Чат не отвечает

```bash
# Перезагрузите страницу (Ctrl+Shift+R)
# Очистите кэш браузера
# Проверьте консоль браузера (F12)
# Проверьте статус LightRAG в правом углу
```

### Проблема: Форматирование не применяется

```bash
# Очистите кэш браузера:
# F12 → Application → Clear site data

# Или используйте Ctrl+Shift+Delete
```

### Проблема: Docker ошибка

```bash
# Пересоберите образ
docker compose down
docker compose build --no-cache
docker compose up

# Или очистите всё и начните заново
docker compose down -v
docker compose up
```
