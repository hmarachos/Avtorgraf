# СЛАЙД 2 — ПРОБЛЕМА

## Боль специалиста на стройке

**Ситуация:**
Подрядчик требует подписать акт скрытых работ на бетон с дефектами. У ГИПа 5 минут до совещания. Нужен точный ответ с нормативным обоснованием.

**Проблемы:**
- ⏱ Ручной поиск в нормативах — **20–40 минут** на один запрос
- 📚 125+ документов СН, СТБ, Кодексов РБ — сложно держать в голове
- 🔗 Нормы перекрёстно связаны — один вопрос затрагивает 3–5 документов одновременно
- ⚠️ Ошибка в документации → штраф Госстройнадзора или лишение аттестата

**Масштаб:** В Беларуси работают тысячи ГИПов и специалистов надзора. Каждый теряет 2–5 часов в день на поиск норм.

# СЛАЙД 3 — РЕШЕНИЕ

## АВТОГРАФ — ответ за 5 секунд с ссылками на пункты НПА

**Что делает система:**
1. Принимает вопрос на естественном языке
2. Ищет ответ по графу знаний из 125+ НПА РБ через LightRAG
3. Возвращает структурированный ответ **со ссылками на конкретные пункты** СН, СТБ, Кодексов

**Ключевые функции:**
- 💬 Чат-интерфейс, адаптированный для смартфона (работает на стройке)
- 🔍 5 режимов поиска: mix, hybrid, global, local, naive
- 👤 3 роли: ГИП, Авторский надзор, Технический надзор
- 💾 История диалогов сохраняется — можно продолжить с любого устройства
- ⚡ Быстрые шаблоны типовых вопросов

**Почему LightRAG, а не обычный RAG:**
- Обычный RAG работает с изолированными документами
- GraphRAG строит **граф связей** между документами → находит перекрёстные нормы

# СЛАЙД 4 — АРХИТЕКТУРА (технический слайд)

## Как устроена система

```
Пользователь (браузер/смартфон)
         ↓ вопрос
АВТОГРАФ Backend (Python, Clean Architecture)
         ↓ GraphRAG запрос
LightRAG Server (GPT-4o + граф знаний)
         ↓ 125+ НПА РБ
         ↑ ответ + ссылки
Пользователь видит ответ за ~5 сек.
```

**Стек:**
- Backend: Python 3.12 (stdlib only, без внешних фреймворков)
- Frontend: Vanilla JS, HTML5, CSS3
- RAG: LightRAG (GraphRAG)
- LLM: OpenAI GPT-4o
- БД: SQLite (история сессий)
- Деплой: Docker + docker-compose
