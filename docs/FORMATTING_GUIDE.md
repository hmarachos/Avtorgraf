# 📘 Гайд по форматированию текста АВТОГРАФ

## Введение

АВТОГРАФ поддерживает **Markdown форматирование** в ответах от модели с **обязательными ссылками на документы**. Это делает ответы более читаемыми и юридически обоснованными.

## 🔗 ГЛАВНОЕ: Ссылки на документы

### ⭐ ОБЯЗАТЕЛЬНО указывайте ссылки

Каждый ответ должен содержать конкретные ссылки на пункты нормативных документов:

```markdown
Согласно **[СН 1.03.01-2019, п. 3.4]** нужны следующие документы...
```

### Формат ссылки

```markdown
**[Название документа, пункт/раздел]**
```

**Примеры:**
- `**[СН 1.03.01-2019, п. 3.4]**` — ссылка на пункт
- `**[СТБ 1937-2022, раздел 2]**` — ссылка на раздел
- `**[Инструкция 1-2020, п. 4.2.1]**` — многоуровневый пункт

**Визуально:** ссылки выделяются подчеркиванием и зеленым цветом

```
✅ **[СН 1.03.01-2019, п. 3.4]** ← Ссылка выделена и подчеркнута
```

## Поддерживаемые элементы

## Поддерживаемые элементы

### 1️⃣ Заголовки

```markdown
# Заголовок уровня 1 (H1)
## Заголовок уровня 2 (H2)
### Заголовок уровня 3 (H3)
```

**Результат:**
- # Заголовок уровня 1
- ## Заголовок уровня 2
- ### Заголовок уровня 3

> **Примечание:** Заголовок уровня 1 отображается меньше в чате, так как это зависит от контекста

---

### 2️⃣ Текстовое форматирование

#### Жирный текст

```markdown
**это жирный текст**
__это тоже жирный текст__
```

**Результат:** **это жирный текст**

---

#### Наклонный текст (курсив)

```markdown
*это наклонный текст*
_это тоже наклонный текст_
```

**Результат:** *это наклонный текст*

---

### 3️⃣ Код

#### Встроенный код

```markdown
Согласно СН 1.03.01-2019, пункт `3.4` определяет процедуру подписания.
```

**Результат:** Согласно СН 1.03.01-2019, пункт `3.4` определяет процедуру подписания.

**Визуально:** код выделяется серым фоном и красным текстом

---

#### Кодовые блоки

```markdown
```
Акт освидетельствования скрытых работ
АОСР-001
Дата: 15.05.2026
```
```

**Результат:**
```
Акт освидетельствования скрытых работ
АОСР-001
Дата: 15.05.2026
```

**Визуально:** весь блок с серым фоном, моноширинный шрифт, горизонтальный скролл если нужен

---

### 4️⃣ Списки

#### Неупорядоченные списки

```markdown
- Первый пункт
- Второй пункт
- Третий пункт
```

**Результат:**
- Первый пункт
- Второй пункт
- Третий пункт

**Визуально:** каждый элемент на новой строке с иконкой (disc)

---

### 5️⃣ Абзацы

**Одиночный перенос строки** — просто новая строка в markdown:

```markdown
Первая строка
Вторая строка
```

**Результат:** Первая строка  
Вторая строка

---

**Двойной перенос строки** — новый абзац:

```markdown
Первый абзац.

Второй абзац.

Третий абзац.
```

**Результат:**

Первый абзац.

Второй абзац.

Третий абзац.

**Визуально:** между абзацами больше пространства

## 🎨 Примеры использования в реальных ответах

### Пример 1: Структурированный ответ

**Входной вопрос:**
> Какие документы нужны для подписания акта освидетельствования скрытых работ?

**Ответ с форматированием:**

```markdown
## Документы для акта освидетельствования скрытых работ

Согласно **СН 1.03.01-2019** при подписании акта освидетельствования скрытых работ (АОСР) необходимо подготовить следующие документы:

### Обязательные документы

- Проектная документация утвержденная органом главного архитектора
- Рабочая документация, согласованная с разработчиком проекта
- Смета затрат на работы (если требуется)
- Акт приемки выполненных работ

### Дополнительные документы

- Сертификаты качества используемых материалов
- Технические паспорта оборудования
- Результаты испытаний и контроля качества
- Журнал выполненных работ с отметками надзирающего лица

### Процедура подписания

1. **Проверка соответствия** — убедитесь что работы выполнены в соответствии с проектом
2. **Документирование** — заполните все поля акта корректно
3. **Подписание** — подпишите акт только если нет дефектов
4. **Регистрация** — сохраните копию в архив проекта

> ⚠️ **Важно:** не подписывайте акт если обнаружены дефекты или отступления от проекта!
```

### Пример 2: Ответ с кодом/примерами

**Входной вопрос:**
> Как правильно заполнить журнал авторского надзора?

**Ответ с форматированием:**

```markdown
## Правильное заполнение журнала авторского надзора

Журнал авторского надзора — это основной документ контроля на строительной площадке.

### Основные требования

Согласно **СН 1.05.01-2019** журнал должен содержать:

- **Дата** — день и месяц выполнения работ
- **Вид работ** — четкое описание (бетонирование, монтаж, отделка)
- **Результаты контроля** — описание обнаруженных дефектов или соответствия норме
- **Подпись ответственного лица** — авторского надзирателя

### Пример записи при обнаружении дефекта

```
Дата: 15.05.2026
Вид работ: Бетонирование фундамента подвала

Обнаружено:
- Неравномерная осадка бетона на участке 2-3 м
- Трещины в опалубке на отметке 1.5 м
- Отклонение геометрии от проекта ±25 мм

Требуется:
1. Переуплотнение бетона вибрацией
2. Проверка опалубки
3. Контроль размеров после набора прочности

Подпись: ____________  Дата: 15.05.2026
```

### Пример записи при соответствии норме

```
Дата: 16.05.2026
Вид работ: Контроль прочности бетона (3 суток)

Результат: Образцы показали прочность 80% от проектной.
В норме. Допускается продолжение работ.

Подпись: ____________  Дата: 16.05.2026
```

### ✅ Лучшие практики

- Записывайте **в день выполнения работ** (не потом!)
- Будьте **объективны** — описывайте факты, а не мнения
- Указывайте **конкретные параметры** (размеры, даты, номера)
- **Требуйте исправления** дефектов в записи
- Всегда **подписывайтесь и проставляйте дату**
```

## 🎯 Советы и трюки

### Совет 1: Комбинирование форматов

```markdown
**Жирный текст с `встроенным кодом` внутри**

*Наклонный текст с **жирным** внутри*
```

### Совет 2: Специальные символы

Используйте эмодзи для визуального выделения:
- ✅ для выполненных пунктов
- ❌ для ошибок или запретов
- ⚠️ для предупреждений
- 📌 для важной информации
- 🔗 для ссылок (хотя не все браузеры поддерживают)

### Совет 3: Горизонтальная линия

```markdown
---
```

Разделяет большие блоки текста визуально.

---

## ⚠️ Что НЕ поддерживается

❌ Таблицы (используйте списки вместо этого)  
❌ Ссылки `[text](url)` (используйте просто текст)  
❌ Изображения `![alt](url)`  
❌ Blockquotes `> цитата`  
❌ Упорядоченные списки с номерами (используйте точки)  
❌ Вложенные списки (только один уровень)
