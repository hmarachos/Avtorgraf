# 🔗 Интеграция ссылок на документы с LightRAG

## 📌 Проблема и решение

### Проблема
- Ответы LightRAG содержали информацию без ссылок на источники
- Было невозможно проверить юридическую обоснованность
- Специалисты не знали на какие нормы ссылаться

### Решение
- **Обновлен системный промпт** чтобы требовать ссылки на документы
- **Обновлен markdown парсер** чтобы выделять ссылки на документы
- **Добавлены CSS стили** для визуального выделения ссылок

---

## 🚀 Как это работает

### 1. Системный промпт (ВАЖНО!)

Файл: `src/avtorgraf/config.py`

```python
SYSTEM_PROMPT = """
Ты помощник проектировщика, авторского и технического надзора 
в Республике Беларусь.

ОБЯЗАТЕЛЬНЫЕ ТРЕБОВАНИЯ К ОТВЕТУ:
1. Используй ТОЛЬКО найденный в LightRAG контекст
2. ВСЕГДА указывай конкретные ссылки на документы:
   - Наименование документа (например: СН 1.03.01-2019)
   - Номер пункта, раздела, статьи (например: пункт 3.4, п. 2.1.5)
3. Формат ссылки должен быть: **[СН 1.03.01-2019, п. 3.4]** или 
   **[СТБ 1937-2022, раздел 4]**
4. Если есть несколько нормативов, перечисли их все
5. Если нет подтверждения в базе - напиши об этом явно
6. НЕ используй нормы РФ, только нормы РБ
7. Каждое утверждение должно иметь ссылку на конкретный пункт документа!
"""
```

### 2. Markdown парсер (app.js)

Парсер ищет ссылки в формате `**[название, пункт]**` и выделяет их:

```javascript
// Ссылки на документы в bold квадратных скобках **[...]**
.replace(/\*\*\[([^\]]+)\]\*\*/g, (match, content) => {
  return `<strong class="reference-link">[${content}]</strong>`;
})
```

### 3. CSS стили (styles.css)

Ссылки выделяются:

```css
.bubble strong.reference-link {
  color: var(--accent);           /* Зеленый цвет */
  border-bottom: 2px solid var(--accent);  /* Подчеркивание */
  cursor: help;                    /* Подсказка при наведении */
}

.bubble strong.reference-link:hover {
  background-color: rgba(36, 93, 77, 0.1);  /* Фон при наведении */
}
```

## 📋 Процесс

```
┌─────────────────────────────────────────────────────────────┐
│ 1. Пользователь задает вопрос                               │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. AVTORGRAF отправляет вопрос + SYSTEM_PROMPT в LightRAG   │
│    (требующий ссылки на документы)                          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. LightRAG возвращает ответ со ссылками                    │
│    "Согласно **[СН 1.03.01-2019, п. 3.4]** нужны..."        │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. Markdown парсер распознает ссылки                         │
│    и оборачивает их в <strong class="reference-link">       │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. CSS стили выделяют ссылки зеленым и подчеркиванием       │
│    Пользователь видит красиво оформленные ссылки ✨         │
└─────────────────────────────────────────────────────────────┘
```

## 🐛 Возможные проблемы и решения

### Проблема: Ссылки отображаются как обычный текст

**Причина:** Парсер не распознал формат `**[...]**`

**Решение:** 
1. Проверьте что в тексте используется формат `**[СН 1.03.01-2019, п. 3.4]**`
2. Очистите кэш браузера (F12 → Application → Clear site data)
3. Перезагрузите страницу

### Проблема: LightRAG не возвращает ссылки

**Причина:** LightRAG не получил или не понял системный промпт

**Решение:**
1. Проверьте что `SYSTEM_PROMPT` правильно установлен в `.env`
2. Перезагрузите приложение
3. Попробуйте другой режим поиска (mix, hybrid, global)

### Проблема: Ссылки на документы, которых нет в базе

**Причина:** LightRAG ссылается на документы которых нет в базе знаний

**Решение:**
1. Загрузите недостающие документы в LightRAG
2. Дождитесь обработки документов (проверьте статус)
3. Задайте вопрос снова
