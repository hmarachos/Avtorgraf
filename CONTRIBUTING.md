# Руководство по внесению вклада в АВТОГРАФ

Спасибо за интерес к проекту АВТОГРАФ! Мы рады любому вкладу в развитие системы.

## 🤝 Как внести вклад

### Сообщить об ошибке

1. Проверьте, не создана ли уже issue с такой же проблемой
2. Создайте новую issue с меткой `bug`
3. Опишите проблему максимально подробно:
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Версия Python и ОС
   - Логи (если есть)

### Предложить улучшение

1. Создайте issue с меткой `enhancement`
2. Опишите предлагаемую функциональность
3. Объясните, какую проблему она решает
4. Приведите примеры использования

### Внести изменения в код

1. **Fork репозитория**
2. **Создайте feature branch:**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Внесите изменения:**
   - Следуйте стилю кода проекта
   - Добавьте docstrings к новым функциям
   - Обновите документацию при необходимости
4. **Проверьте код:**
   ```bash
   # Форматирование
   black src/
   isort src/
   
   # Линтинг
   ruff check src/
   
   # Типы
   mypy src/
   ```
5. **Commit изменений:**
   ```bash
   git commit -m "feat: добавлена новая функция X"
   ```
6. **Push в ваш fork:**
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Создайте Pull Request**

## 📝 Стиль кода

### Python

- Используйте **Python 3.12+**
- Следуйте **PEP 8**
- Используйте **type hints** для всех функций
- Максимальная длина строки: **100 символов**
- Используйте **docstrings** в формате Google:

```python
def function_name(param1: str, param2: int) -> bool:
    """
    Краткое описание функции.
    
    Args:
        param1: Описание первого параметра
        param2: Описание второго параметра
        
    Returns:
        Описание возвращаемого значения
        
    Raises:
        ValueError: Когда возникает эта ошибка
    """
    pass
```

### JavaScript

- Используйте **ES6+**
- Предпочитайте `const` над `let`
- Используйте async/await вместо промисов
- Именуйте переменные описательно

### HTML/CSS

- Семантичная разметка
- Доступность (ARIA атрибуты)
- Mobile-first подход

## 🏗️ Архитектура

Проект следует принципам **Clean Architecture**:

- **domain/** — чистые доменные модели без зависимостей
- **application/** — бизнес-логика, оркестрация
- **infrastructure/** — работа с БД, внешними API
- **presentation/** — HTTP API, веб-интерфейс

### Принципы

- Dependency Inversion (зависимости направлены внутрь)
- Single Responsibility (один класс = одна ответственность)
- Immutability (используем `@dataclass(frozen=True)`)
- No external dependencies (только stdlib Python)

## ✅ Checklist для Pull Request

- [ ] Код следует стилю проекта
- [ ] Добавлены docstrings к новым функциям/классам
- [ ] Обновлена документация (README.md)
- [ ] Проверка типов (mypy) проходит успешно
- [ ] Линтер (ruff) не выдает ошибок
- [ ] Код отформатирован (black, isort)
- [ ] Commit messages следуют [Conventional Commits](https://www.conventionalcommits.org/)

## 📦 Типы коммитов

- `feat:` — новая функциональность
- `fix:` — исправление бага
- `docs:` — изменения в документации
- `style:` — форматирование, отступы (не влияет на код)
- `refactor:` — рефакторинг без изменения функциональности
- `perf:` — улучшение производительности
- `test:` — добавление тестов
- `chore:` — изменения в сборке, CI/CD

## 🧪 Тестирование

В данный момент проект не имеет автоматических тестов. Вклад в создание тестов приветствуется!

### Ручное тестирование

Перед созданием PR проверьте:

1. **Запуск приложения:**
   ```bash
   PYTHONPATH=src python -m avtorgraf.main
   ```

2. **Health check:**
   ```bash
   curl http://localhost:8080/api/health
   ```

3. **Отправка вопроса:**
   ```bash
   curl -X POST http://localhost:8080/api/chat \
     -H "Content-Type: application/json" \
     -d '{"question":"Тестовый вопрос","mode":"mix"}'
   ```

4. **Веб-интерфейс:**
   - Откройте http://localhost:8080
   - Проверьте все режимы поиска
   - Протестируйте на разных устройствах
