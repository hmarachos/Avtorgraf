# 📋 Changelog — версия 2.0.0 (2026-06-02)

## ✨ Что нового

### 🎨 Фронтенд

#### Форматирование ответов модели ✅
- **Добавлен встроенный markdown парсер** для преобразования ответов в форматированный HTML
- **Поддерживаемые элементы:**
  - Заголовки: `# H1`, `## H2`, `### H3`
  - Текстовое форматирование: **жирный**, *наклонный*
  - Встроенный код: `` `code` ``
  - Кодовые блоки: ` ```code block``` `
  - Списки: `- item 1`
  - Абзацы: автоматическое разделение по двойному Enter

#### CSS Стили ✅
- Добавлены специальные стили для отображения форматированного текста
- Улучшена читаемость через:
  - Правильные отступы между абзацами
  - Индентация списков
  - Выделение кода цветом и шрифтом
  - Блоки кода с фоном

#### UI/UX улучшения ✅
- Статус LightRAG теперь показывает ✓ или ✗
- Улучшены сообщения об ошибках с эмодзи
- Источники отображаются с дополнительным источником (source)
- Время выполнения запроса отделено горизонтальной линией

### 🏗️ Backend

#### Архитектура `ChatService` ✅
- **Выделены отдельные методы** для каждой ответственности:
  - `_format_history()` — форматирование истории для LightRAG
  - `_extract_answer()` — извлечение текста ответа
  - `_extract_references()` — извлечение нормативных ссылок
  - `_parse_reference_item()` — парсинг отдельной ссылки

#### Архитектура `HttpServer` ✅
- **Выделены методы-обработчики** для каждого эндпоинта:
  - `_handle_health()` — проверка здоровья
  - `_handle_sessions()` — список сессий
  - `_handle_documents()` — список документов
  - `_handle_pipeline_status()` — статус обработки
  - `_handle_chat()` — обработка чата
  - `_format_response()` — форматирование ответа

#### Обработка ошибок ✅
- Улучшена обработка исключений с правильными HTTP статусами
- `ValueError` → 400 Bad Request
- `LightRagError` → 502 Bad Gateway
- Все остальные исключения → 500 Internal Server Error
- Каждый обработчик имеет try-except блок

### 📝 Документация

- **Добавлены JSDoc комментарии** во все JavaScript функции
- **Добавлены Docstrings** во все Python методы
- **Создан файл `REFACTORING.md`** с подробным описанием всех изменений
- **Улучшен `README.md`** с полной документацией

## 🚀 Как обновиться

1. **Обновите файлы из репозитория:**
   ```bash
   git pull origin main
   ```

2. **Перезагрузите приложение:**
   - Локальный запуск: `Ctrl+C` + заново
   - Docker: `docker compose restart`

3. **Очистите кэш браузера:**
   - Откройте DevTools (F12)
   - Application → Clear site data

## 🐛 Известные ограничения

- Markdown парсер не поддерживает: таблицы, ссылки, изображения
- Для более полной поддержки рассмотрите использование `marked.js`
- Подсветка синтаксиса для кода требует дополнительной библиотеки `highlight.js`
